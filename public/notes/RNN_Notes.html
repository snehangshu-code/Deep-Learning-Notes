01 — Introduction

What is a Recurrent Neural Network?

A Recurrent Neural Network (RNN) is a class of neural network designed to work with sequential data — data where order matters. Unlike a standard feedforward network that processes each input independently, an RNN maintains a hidden state (memory) that carries information from previous time steps forward.

Intuition Think of reading a sentence word by word. When you reach the word "it", your brain uses context from earlier words to understand what "it" refers to. RNNs do exactly this — they remember context as they move through a sequence.

Why not a standard feedforward network?

Feedforward networks have fixed-size inputs and no memory. For a sequence of length 100, you'd need 100 separate input neurons and couldn't share learned patterns across positions. RNNs solve this by using the same weights at every time step, with a hidden state that evolves over time.

02 — Architecture

The RNN Architecture

An RNN consists of a recurrent cell that takes two inputs at each time step: the current input xₜ and the previous hidden state h_{t-1}, and produces a new hidden state hₜ and (optionally) an output ŷₜ.

03 — Forward Propagation

Forward Propagation

Forward propagation in an RNN processes the sequence one time step at a time, from left to right. At each step, two computations happen:

  1. Compute the new hidden state hₜ using the current input and the previous hidden state.
  2. Compute the output ŷₜ from the current hidden state.
— HIDDEN STATE UPDATE —
hₜ = tanh( Wₓₕ · xₜ + Wₕₕ · h₍ₜ₋₁₎ + bₕ )

— OUTPUT COMPUTATION —
ŷₜ = softmax( Wₕᵧ · hₜ + bᵧ )

— LOSS AT EACH STEP (cross-entropy) —
Lₜ = − yₜ · log(ŷₜ) // yₜ = true label

— TOTAL LOSS —
L = Σ Lₜ // sum over all T time steps
Fig 2 — Single RNN Cell: Forward Pass Detail
Key Insight The weights Wₓₕ, Wₕₕ, and Wₕᵧ are shared across all time steps. This is what gives RNNs the ability to handle variable-length sequences and generalize learned patterns across positions.

Initialization

At the very first time step (t = 1), there is no previous hidden state, so we initialize h₀ = 0 (a zero vector). Some implementations learn h₀ as a parameter too.

04 — Backward Propagation

Backpropagation Through Time (BPTT)

Training an RNN requires computing gradients of the total loss L with respect to all weight matrices (Wₓₕ, Wₕₕ, Wₕᵧ). Because the hidden state at each step depends on all previous steps, we must unroll the network through time and apply the chain rule — this algorithm is called Backpropagation Through Time (BPTT).

Fig 3 — BPTT: Gradient Flow (Right to Left)

Step-by-Step BPTT Derivation

Let's derive the key gradient step by step. The total loss is L = Σ Lₜ. We want ∂L/∂Wₕₕ.

  1. Gradient of output layer
    At each step t: ∂Lₜ/∂Wₕᵧ = ∂Lₜ/∂ŷₜ · ∂ŷₜ/∂Wₕᵧ
    This is simple — no time dependency here.
  2. Gradient of hidden state
    ∂Lₜ/∂hₜ = Wₕᵧᵀ · ∂Lₜ/∂ŷₜ (from output loss)
    + ∂L₍ₜ₊₁₎/∂hₜ (from future steps — the recurrent part!)
  3. Backprop through tanh
    ∂hₜ/∂aₜ = (1 − hₜ²) where aₜ = Wₓₕxₜ + Wₕₕh₍ₜ₋₁₎ + bₕ
    This is the tanh derivative: tanh'(x) = 1 − tanh²(x)
  4. Gradient of Wₕₕ
    ∂L/∂Wₕₕ = Σₜ [ δₜ · h₍ₜ₋₁₎ᵀ ]
    where δₜ = ∂L/∂hₜ · (1 − hₜ²) — accumulated delta at step t
  5. Update weights
    Wₕₕ ← Wₕₕ − η · ∂L/∂Wₕₕ (same for Wₓₕ, Wₕᵧ)
— BPTT CORE RECURRENCE (delta) —
δₜ = ( Wₕₕᵀ · δ₍ₜ₊₁₎ + Wₕᵧᵀ · ∂Lₜ/∂ŷₜ ) ⊙ (1 − hₜ²)

— WEIGHT GRADIENTS (summed over T steps) —
∂L/∂Wₕₕ = Σₜ δₜ · h₍ₜ₋₁₎ᵀ
∂L/∂Wₓₕ = Σₜ δₜ · xₜᵀ
∂L/∂Wₕᵧ = Σₜ ∂Lₜ/∂ŷₜ · hₜᵀ
Fig 4 — Gradient Signal Through a Single Cell
Truncated BPTT For very long sequences, computing gradients all the way back to t=1 is expensive and unstable. Truncated BPTT limits backpropagation to a fixed window of k steps, trading accuracy for speed and stability.
05 — Training Challenges

Vanishing & Exploding Gradients

The most notorious problem with RNNs. During BPTT, the gradient for step t involves multiplying Wₕₕ repeatedly:

— GRADIENT THROUGH T STEPS —
∂hₜ/∂h₁ = Π_{k=1}^{t-1} ( Wₕₕᵀ · diag(1 − hₖ²) )

// This is a product of T-1 matrices — grows or shrinks exponentially!
Fig 5 — Vanishing Gradient: Signal Fades Over Time
ProblemCauseEffectFix
Vanishing gradient Eigenvalues of Wₕₕ < 1 → product → 0 Model forgets long-range dependencies; early layers don't learn LSTM / GRU, gradient clipping
Exploding gradient Eigenvalues of Wₕₕ > 1 → product → ∞ NaN losses, unstable training Gradient clipping (if ‖g‖ > threshold, rescale g)
Gradient Clipping A simple fix for exploding gradients: if the gradient norm exceeds a threshold τ, rescale it.
g ← g · (τ / max(τ, ‖g‖))
